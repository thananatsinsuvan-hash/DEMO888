# ระบบบันทึกค่าใช้จ่ายผู้สูงอายุ — เวอร์ชัน Cloudflare D1 + Login

โครงสร้างโปรเจกต์:

```
.
├── schema.sql            ตารางฐานข้อมูล D1 (รวมตาราง users สำหรับ login)
├── migrations/
│   └── 002_case_details.sql  migration เพิ่มฟิลด์ประวัติเคส/รูป (สำหรับฐานข้อมูลที่ deploy ไปแล้ว)
├── wrangler.toml          การตั้งค่า Worker + binding ไป D1/Assets/R2
├── package.json
├── scripts/
│   └── create-user.js       สคริปต์สร้างผู้ใช้ใหม่ (คำนวณ hash รหัสผ่านให้) — สำรองไว้ใช้ผ่าน CLI
├── src/
│   └── index.js              Worker API — login/session + ข้อมูลตามสิทธิ์สาขา + จัดการผู้ใช้ + อัปโหลดรูป
└── public/
    └── index.html             หน้าเว็บระบบ มีหน้า login ในตัว + หน้าจัดการผู้ใช้ (admin)
```

## ฟีเจอร์ล่าสุด (การเงิน/บิล/PWA)

- **บัญชีธนาคารต่อสาขา** — แต่ละสาขาใส่บัญชีธนาคารได้หลายบัญชี (แท็บ "ตั้งค่าเอกสาร" > แก้ไขสาขา) แล้ว staff เลือกบัญชีที่จะใช้เรียกเก็บได้ตอนเพิ่ม/แก้ไขเคส
- **โลโก้ศูนย์แบบอัปโหลดไฟล์** — อัปโหลดรูปโลโก้จริงได้ที่ฟอร์มองค์กร (เก็บบน R2 เหมือนรูปคนไข้) ใช้แสดงบนหัวใบแจ้งยอด/ใบเสร็จแทนตัวย่อ ถ้ายังไม่อัปโหลดจะใช้ตัวย่อโลโก้เดิมแทน
- **ค่าบริการรายเดือนแสดงช่วงวันที่ล่วงหน้า** — เช่น "ค่าบริการประจำเดือน วันที่ 15 ก.ค. - 14 ส.ค. 69" คำนวณจากวันครบกำหนดชำระของเคสอัตโนมัติ
- **ประเภทรายการค่าใช้จ่ายอื่นๆ แบบเลือกได้** — ค่ามัดจำการให้บริการ, ค่าบริการหัตถการทางการพยาบาล (ระบุเพิ่ม), ค่าบริการเจ้าหน้าที่พาไปพบแพทย์, ค่าบริการจัดหาเวชภัณฑ์ (ระบบบวก service charge 10% ให้อัตโนมัติเป็นอีกหนึ่งรายการ), อื่นๆ (ระบุ)
- **แท็บ "บิลค้างชำระ"** — staff เห็นเฉพาะบิลที่ยังไม่ชำระของสาขาตัวเอง (ไม่ต้องดูสรุปยอดทั้งหมด) และยืนยันการชำระเงินได้ พร้อมแนบรูปสลิปโอนเงิน/หลักฐานรับเงินสด (แท็บ "สรุปรายเดือน" เดิมเป็นของ admin เท่านั้น)
- **ส่วนลดในใบเสร็จ** — ใส่ยอดส่วนลด (บาท) ตอนบันทึกการชำระเงินได้ ใบเสร็จจะแสดงยอดก่อนหักส่วนลด/ส่วนลด/ยอดสุทธิ พร้อมแนบรูปสลิปที่อัปโหลดไว้
- **ติดตั้งเป็นแอปบนหน้าจอโทรศัพท์ได้ (PWA)** — เปิดเว็บแล้วเลือก "เพิ่มลงหน้าจอโฮม" ได้ทั้ง iOS/Android
- **แจ้งเตือน admin เมื่อถึงวันครบกำหนด/เกินกำหนดชำระ** — ขึ้น banner ในเว็บทันทีที่ admin ล็อกอิน (ปิดแจ้งเตือนได้ถึงพรุ่งนี้)
- **เลือกได้ว่าจะรวมค่าบริการประจำเดือนในบิลเดือนนั้นหรือไม่** — มีติ๊กถูกที่หน้า "บันทึกค่าใช้จ่าย" ต่อเคส/ต่อเดือน ค่าเริ่มต้นคือรวมเสมอ
- **ส่วนลดในใบเสร็จหักออกจากยอดจริง** — กรอกส่วนลดแล้วยอดสุทธิในใบเสร็จจะลดลงตามนั้นทันที
- **ใบแจ้งยอด/ใบเสร็จพอดีกระดาษ A4** — ปรับระยะขอบและขนาดตัวอักษรตอนพิมพ์ให้อยู่ในหน้าเดียว
- **บิลพิเศษระหว่างเดือน** — สร้างบิลเรียกเก็บแยกจากรอบบิลรายเดือนได้ (เช่น เรียกเก็บค่าเวชภัณฑ์อย่างเดียวกลางเดือน) มีเลขที่เอกสารและสถานะการชำระของตัวเอง ไม่กระทบยอด/สถานะของค่าบริการประจำเดือน จัดการได้จากหน้า "บันทึกค่าใช้จ่าย" ของแต่ละเคส และแสดงรวมอยู่ในแท็บ "บิลค้างชำระ" ด้วย
- **รวม service charge ค่าเวชภัณฑ์เป็นรายการเดียว** — ถ้าเดือนนั้น/บิลนั้นมีรายการ "ค่าบริการจัดหาเวชภัณฑ์" หลายรายการ ระบบจะรวมยอดเวชภัณฑ์ทั้งหมดก่อน แล้วคิด service charge 10% เพียงครั้งเดียว (ยอดรวมเท่าเดิม แต่แสดงเป็นบรรทัดเดียวแทนที่จะซ้ำกันหลายบรรทัด) ใช้กับทั้งบิลรายเดือนและบิลพิเศษระหว่างเดือน
- **แยกดูตามสาขาได้ในแท็บ "บิลค้างชำระ" และ "สรุปรายเดือน"** — มีตัวเลือก "ทุกสาขา" หรือเลือกสาขาใดสาขาหนึ่งโดยเฉพาะ (ตัวเลือกจะแสดงเฉพาะสาขาที่ผู้ใช้คนนั้นเข้าถึงได้)
- **เลขที่เอกสารรันอัตโนมัติแบบเป็นระบบ** — ใบแจ้งยอด (INV) และใบเสร็จ (RCT) รันเลขต่อเนื่องแยกชุดกัน แยกตามสาขา และรีเซ็ตกลับเป็น 0001 ทุกต้นปี พ.ศ. เช่น `INV-NW-2569-0001`, `RCT-NW-2569-0001` (ดูรายละเอียดหัวข้อ "ระบบเลขที่เอกสาร" ด้านล่าง)
- **บันทึก/แชร์ใบแจ้งยอด-ใบเสร็จเป็นไฟล์ JPG** — ในหน้าต่างแสดงเอกสาร มีปุ่ม "บันทึก/แชร์รูปภาพ (JPG)" นอกเหนือจากปุ่มพิมพ์/บันทึก PDF เดิม เหมาะสำหรับส่งเอกสารผ่าน LINE/แชทโดยไม่ต้องพิมพ์เป็นกระดาษก่อน — **บนมือถือ** (iOS/Android ที่รองรับ Web Share API) ปุ่มนี้จะเปิดหน้าต่างแชร์ของเครื่องให้เลือก "บันทึกรูปภาพ"/"Save to Photos" เพื่อเก็บเข้า Gallery ได้โดยตรง ถ้าเครื่องไม่รองรับจะดาวน์โหลดเป็นไฟล์ไปที่โฟลเดอร์ดาวน์โหลดของเบราว์เซอร์แทน (บนคอมพิวเตอร์จะดาวน์โหลดเป็นไฟล์ตามปกติเสมอ)
- **แท็บ "สรุปเอกสาร"** — สรุปรายการใบแจ้งยอด/ใบเสร็จที่ออกเลขที่เอกสารแล้วทั้งหมดในเดือนที่เลือก แยกตามสาขา (เห็นได้เฉพาะ admin/manager) ใบแจ้งยอดอิงตามรอบบิล/วันที่ออกบิล ส่วนใบเสร็จอิงตามวันที่ชำระเงินจริง พร้อมยอดรวมแต่ละส่วน
- **จัดการเคสที่ย้ายออก** — ในหน้า "รายชื่อเคส" กดปุ่ม "ย้ายออก" ที่การ์ดเคส แล้วระบุวันที่ย้ายออก เคสจะถูกย้ายไป "คลังเก็บถาวร" ทันที (ไม่นับรวมในบิลค้างชำระ/สรุปรายเดือน/แจ้งเตือนครบกำหนดอีกต่อไป แต่ข้อมูลเดิมยังเก็บไว้ครบ) กดปุ่ม "คลังเก็บถาวร (เคสย้ายออก)" เพื่อดูรายชื่อเคสที่ย้ายออกแล้ว และกด "กู้คืน" เพื่อนำเคสกลับมาใช้งานได้ทุกเมื่อ
- **แก้ไขรายการค่าใช้จ่ายที่เพิ่มไปแล้ว** — ในหน้า "บันทึกค่าใช้จ่าย" แต่ละรายการมีปุ่ม "แก้ไข" (ยกเว้นรายการ service charge เวชภัณฑ์ที่คำนวณอัตโนมัติ) กดแล้วฟอร์มด้านล่างจะดึงค่าเดิมมาให้แก้ไขได้ทันที
- **รวมค่าบริการพาไปพบแพทย์หลายวันเป็นบรรทัดเดียวในเอกสาร** — บันทึกแต่ละครั้งที่พาไปพบแพทย์แยกเป็นรายวันได้ตามปกติในหน้าบันทึกค่าใช้จ่าย แต่ตอนออกใบแจ้งยอด/ใบเสร็จ ถ้ามีมากกว่า 1 ครั้งในเดือนเดียวกัน ระบบจะรวมยอดและแสดงเป็นบรรทัดเดียว เช่น "ค่าบริการเจ้าหน้าที่พาไปพบแพทย์ — วันที่ 2, 15 และ 25 เม.ย. 69 จำนวน 3 วัน"
- **แท็บ "ปฏิทินครบกำหนด"** — ภาพรวมแบบปฏิทินรายเดือน แสดงว่าเคสไหนครบกำหนดชำระวันไหน (สีตามสถานะ: รอชำระ/เกินกำหนด/ชำระแล้ว) เห็นได้ทุกบทบาท (admin/manager/staff) โดยแสดงเฉพาะเคสในสาขาที่ตนเองมีสิทธิ์เข้าถึงเท่านั้น กดที่ชื่อเคสในปฏิทินเพื่อเปิดฟอร์มยืนยันการชำระได้ทันที
- **บันทึกนัดหมอในปฏิทิน** — กดปุ่ม "+ เพิ่มนัดหมอ" ในแท็บปฏิทิน เลือกเคส/วันที่/รายละเอียดนัด แล้วนัดจะไปแสดงเป็นชิปในวันนั้นบนปฏิทิน กดที่ชิปเพื่อแก้ไข "ยกเลิกนัดนี้" (เปลี่ยนสถานะ เก็บประวัติไว้ ไม่ลบ) "กู้คืนนัดนี้" หรือ "ลบนัดนี้ถาวร" ก็ได้
- **แจ้งเตือนลงค่าใช้จ่ายพาไปพบแพทย์** — ถ้าถึง/เลยวันนัดแล้วแต่ยังไม่ได้บันทึกรายการ "ค่าบริการเจ้าหน้าที่พาไปพบแพทย์" ของวันนั้นในหน้าบันทึกค่าใช้จ่าย ระบบจะขึ้น banner เตือนทันทีที่ล็อกอิน (เห็นได้ทุกบทบาทตามสิทธิ์สาขาของตัวเอง ไม่ใช่ admin เท่านั้น) ปิดแจ้งเตือนได้ถึงพรุ่งนี้ แจ้งเตือนจะหายไปเองเมื่อบันทึกค่าใช้จ่ายของวันนั้นแล้ว

## ข้อมูลประวัติเคส (ล่าสุด)

แต่ละเคสมีข้อมูลเพิ่มเติมนอกจากค่าบริการ:
- **ประวัติเบื้องต้น** — อายุ, โรคประจำตัว/ข้อมูลสุขภาพสำคัญ, ยาที่แพ้, ผู้ติดต่อฉุกเฉิน+เบอร์โทร
- **วันที่เข้ารับบริการ** — แทนที่การกรอก "วันครบกำหนดชำระ" ด้วยมือ ระบบจะคำนวณวันครบกำหนดชำระของทุกเดือนจากวันที่นี้อัตโนมัติ (ใช้ day-of-month ของวันที่เข้ารับบริการ)
- **รูปคนไข้** — อัปโหลดได้จากฟอร์มเคส เก็บไฟล์จริงบน Cloudflare R2 (จำกัดไฟล์ไม่เกิน 5MB ต้องเป็นไฟล์รูปภาพ)

ต้องตั้งค่าเพิ่มเติมก่อนใช้งานฟีเจอร์นี้ (ดูขั้นตอนที่ 2.1 และ 3.1 ด้านล่าง)

## ระบบเลขที่เอกสาร

เลขที่ใบแจ้งยอด/ใบเสร็จ **ออกโดยเซิร์ฟเวอร์เท่านั้น** (ไม่ใช่สุ่มจากฝั่งเว็บเหมือนเดิม) เพื่อให้รันต่อเนื่องจริงและไม่ซ้ำแม้เปิดพร้อมกันหลายเครื่อง:

- **แยกรันตามสาขา** — แต่ละสาขามีชุดเลขของตัวเอง ไม่ปนกับสาขาอื่น
- **แยกชุดกันระหว่างใบแจ้งยอดกับใบเสร็จ** — `INV-...` กับ `RCT-...` คนละชุด คนละเลขรัน
- **รีเซ็ตทุกปี พ.ศ.** — ขึ้นปีใหม่ เลขรันของทุกสาขา/ทุกประเภทจะเริ่มนับจาก 0001 ใหม่โดยอัตโนมัติ
- รูปแบบ: `INV-{รหัสสาขา}-{ปี พ.ศ.}-{เลขรัน 4 หลัก}` เช่น `INV-NW-2569-0001`
- เลขที่จะถูก "จอง" ครั้งแรกที่เปิดดูเอกสารนั้น (ใบแจ้งยอด) หรือตอนกดยืนยันการชำระเงิน (ใบเสร็จ) แล้วบันทึกลงฐานข้อมูลถาวร เปิดดูซ้ำจะได้เลขเดิมเสมอ ไม่รันเลขใหม่ทุกครั้ง

**รหัสสาขา**: ตั้งได้ที่แท็บ "ตั้งค่าเอกสาร" > แก้ไขสาขา > ช่อง "รหัสสาขา" (เช่น `NW`, `BKK1`) ถ้าไม่ตั้งไว้ระบบจะสร้างรหัสสำรองให้อัตโนมัติจาก ID ภายในของสาขา — แนะนำให้ตั้งรหัสสาขาที่มีความหมายไว้ตั้งแต่แรก เพราะเลขที่เอกสารที่ออกไปแล้วจะไม่เปลี่ยนตามถ้าแก้รหัสสาขาภายหลัง (มีผลกับเอกสารที่ออกใหม่เท่านั้น)

## ระบบสิทธิ์การเข้าถึง

มี 3 บทบาท:
- **admin (ผู้ดูแลระบบ)** — เห็น/แก้ไขได้ทุกองค์กร ทุกสาขา ทุกเคส จัดการองค์กร/สาขาได้ (แท็บ "ตั้งค่าเอกสาร") เพิ่ม/ลบผู้ใช้และรีเซ็ตรหัสผ่านได้เองผ่านแท็บ "จัดการผู้ใช้" และเป็นผู้กำหนดว่า staff/manager แต่ละคนเข้าถึงสาขาใดได้บ้าง
- **manager (ผู้จัดการ)** — เห็น/แก้ไขข้อมูลได้เฉพาะสาขาที่ admin อนุญาตเท่านั้น เลือกได้**หลายสาขาพร้อมกัน** เห็นทั้งแท็บ "บิลค้างชำระ" และ "สรุปรายเดือน"/"สรุปเอกสาร" ของสาขาที่ตัวเองเข้าถึงได้ แต่ไม่เห็นแท็บ "ตั้งค่าเอกสาร" / "จัดการผู้ใช้"
- **staff (พนักงานสาขา)** — เห็น/แก้ไขข้อมูลได้เฉพาะสาขาที่ admin อนุญาตเท่านั้นเช่นเดียวกับ manager (**เลือกได้หลายสาขาพร้อมกัน** ไม่ได้จำกัดแค่สาขาเดียวอีกต่อไป) เพิ่ม/แก้ไข/ลบเคสและค่าใช้จ่ายได้เฉพาะในสาขาที่เข้าถึง มองไม่เห็นและแก้ไขสาขาอื่นไม่ได้เลย แต่ไม่เห็นแท็บ "สรุปรายเดือน" / "สรุปเอกสาร" / "ตั้งค่าเอกสาร" / "จัดการผู้ใช้" (ต่างจาก manager ตรงนี้)

ทุกบทบาท (ยกเว้น admin) ถูกบังคับสิทธิ์ฝั่งเซิร์ฟเวอร์เสมอ (ตรวจ branch_id/user_branch_access ทุกครั้งก่อนอ่าน/เขียนข้อมูลจริง) ไม่ใช่แค่ซ่อนแท็บที่หน้าเว็บ

## หน้าจัดการผู้ใช้ (admin เท่านั้น)

จากแท็บ **จัดการผู้ใช้** ในเว็บ ผู้ดูแลระบบสามารถ:
- ดูรายชื่อผู้ใช้ทั้งหมด พร้อมบทบาทและรายชื่อสาขาที่เข้าถึงได้ทั้งหมด (staff และ manager แสดงแบบเดียวกัน)
- **เพิ่มผู้ใช้ใหม่** — กรอก username / รหัสผ่าน / ชื่อที่แสดง / เลือกบทบาท (admin, manager หรือ staff) แล้วถ้าเป็น manager หรือ staff ให้ติ๊กเลือกสาขาที่เข้าถึงได้ (จัดกลุ่มตามองค์กร เลือกได้หลายสาขา) อย่างน้อย 1 สาขา
- **จัดการสิทธิ์** (สำหรับผู้ใช้ role manager หรือ staff) — แก้ไขรายการสาขาที่เข้าถึงได้ภายหลัง โดยไม่ต้องลบ/สร้างผู้ใช้ใหม่ — ใช้ปุ่มนี้เพื่อเพิ่ม/ลดจำนวนสาขาที่ staff คนหนึ่งดูแลได้ตลอดเวลา
- **ลบผู้ใช้** — ลบได้ทันที (ยกเว้นบัญชีที่กำลังล็อกอินอยู่ ป้องกันการลบตัวเองพลาด)
- **รีเซ็ตรหัสผ่าน** — ตั้งรหัสผ่านใหม่ให้ผู้ใช้คนใดก็ได้

**หมายเหตุสำหรับบัญชี staff ที่สร้างไว้ก่อนอัปเดตนี้**: บัญชีเก่ายังใช้งานได้ตามปกติ (ยังผูกกับสาขาเดี่ยวเดิมอยู่) ถ้าต้องการให้ดูได้หลายสาขา ให้กด "จัดการสิทธิ์" แล้วติ๊กเพิ่มสาขาที่ต้องการได้เลย ไม่ต้องลบบัญชีเดิมทิ้ง

ระบบตรวจสอบสิทธิ์ทั้งฝั่งเว็บ (ซ่อนแท็บถ้าไม่ใช่ admin) และฝั่งเซิร์ฟเวอร์ (ปฏิเสธคำขอถ้า role ไม่ใช่ admin) และใช้วิธี hash รหัสผ่านแบบเดียวกับ `scripts/create-user.js` (PBKDF2-SHA256) จึงล็อกอินได้ทันทีไม่ว่าจะสร้างผ่านเว็บหรือ CLI

สคริปต์ `scripts/create-user.js` ยังใช้งานได้อยู่เป็นทางเลือกสำรอง (เผื่อกรณีตั้งค่า admin คนแรกก่อนมีบัญชี admin ในระบบ)

## วิธีติดตั้งและรัน

### 1) เตรียมเครื่องมือ
```bash
npm install -g wrangler
wrangler login
npm install
```

### 2) สร้างฐานข้อมูล D1
```bash
wrangler d1 create care-billing-db
```
คัดลอกค่า `database_id` ที่ได้ไปแทนที่ `PASTE_YOUR_DATABASE_ID_HERE` ใน `wrangler.toml`

### 2.1) สร้าง R2 bucket สำหรับเก็บรูปคนไข้
```bash
wrangler r2 bucket create care-billing-photos
```
ชื่อ bucket ต้องตรงกับ `bucket_name` ใน `wrangler.toml` (ตั้งไว้เป็น `care-billing-photos` แล้ว ไม่ต้องแก้อะไรเพิ่ม)

### 3) สร้างตารางในฐานข้อมูล
```bash
npm run db:init:remote
```
ถ้าเป็นฐานข้อมูลที่เคยรันคำสั่งนี้ไปแล้วก่อนหน้า (มีข้อมูลอยู่แล้ว) ให้รัน migration เพิ่มคอลัมน์ประวัติเคส/รูปด้วย:
```bash
npm run db:migrate:002
```
และ migration เพิ่มบัญชีธนาคารต่อสาขา/โลโก้อัปโหลดได้/ประเภทรายการ/ส่วนลด-สลิปโอนเงิน:
```bash
npm run db:migrate:003
```
และ migration เพิ่มการเปิด/ปิดค่าบริการประจำเดือนรายเดือน:
```bash
npm run db:migrate:004
```
และ migration เพิ่มบิลพิเศษระหว่างเดือน:
```bash
npm run db:migrate:005
```
และ migration เพิ่มบทบาท manager (ตาราง `user_branch_access` สำหรับกำหนดสิทธิ์หลายสาขา):
```bash
npm run db:migrate:006
```
และ migration เพิ่มระบบเลขที่เอกสารรันอัตโนมัติ (คอลัมน์ `code` ในตาราง branches + ตาราง `doc_sequences`):
```bash
npm run db:migrate:007
```
และ migration เพิ่มการจัดการเคสที่ย้ายออก (คอลัมน์ `moved_out`, `move_out_date` ในตาราง cases):
```bash
npm run db:migrate:008
```
และ migration เพิ่มตารางนัดหมอ (สำหรับแท็บปฏิทิน + แจ้งเตือนลงค่าใช้จ่าย):
```bash
npm run db:migrate:009
```

### 4) ตั้งค่า secret สำหรับเซ็น session (สำคัญ ห้ามข้าม)
```bash
wrangler secret put SESSION_SECRET
```
ระบบจะถามค่า — พิมพ์ข้อความสุ่มยาวๆ อะไรก็ได้ (เก็บเป็นความลับ ไม่ต้องจำ ใช้ครั้งเดียวตอนตั้งค่า)

### 5) Deploy ก่อน แล้วค่อยสร้างผู้ใช้
```bash
npm run deploy
```

### 6) สร้างบัญชี admin คนแรก (ผ่าน CLI เท่านั้น เพราะยังไม่มีใครล็อกอินได้)
```bash
node scripts/create-user.js admin1 "รหัสผ่านที่ต้องการ" "ผู้ดูแลระบบ" admin
```
คำสั่งนี้จะพิมพ์ SQL ออกมา — คัดลอกไปรัน:
```bash
wrangler d1 execute care-billing-db --remote --command="INSERT INTO users (...) VALUES (...)"
```
(หรือบันทึกเป็นไฟล์ `seed-users.sql` แล้วรัน `wrangler d1 execute care-billing-db --remote --file=./seed-users.sql`)

จากนั้นเข้าเว็บ ล็อกอินด้วยบัญชี admin นี้ แล้วไปที่แท็บ **ตั้งค่าเอกสาร** เพื่อเพิ่มองค์กรและสาขาให้ครบตามต้องการ

### 7) สร้างผู้ใช้ manager/staff คนต่อๆ ไป — ทำผ่านเว็บได้เลย
เข้าเว็บด้วยบัญชี admin → แท็บ **จัดการผู้ใช้** → กด "+ เพิ่มผู้ใช้" → เลือกบทบาท (manager หรือ staff) → ติ๊กเลือกสาขาที่เข้าถึงได้ (เลือกได้หลายสาขา ต้องสร้างสาขาไว้ในแท็บ "ตั้งค่าเอกสาร" ก่อน) → บันทึก (ทำซ้ำได้ตามจำนวนที่ต้องการ)

### 8) ทดสอบ
เปิด URL ของระบบ ล็อกอินด้วยบัญชี staff ที่สร้างไว้ → ควรเห็นเฉพาะเคส/ข้อมูลของสาขาที่ผูกไว้เท่านั้น และไม่เห็นแท็บ "ตั้งค่าเอกสาร" / "จัดการผู้ใช้"

## การจัดการผู้ใช้ภายหลัง

- **เปลี่ยนรหัสผ่าน**: เข้าเว็บด้วยบัญชี admin → แท็บ "จัดการผู้ใช้" → กด "รีเซ็ตรหัสผ่าน" ที่ผู้ใช้คนนั้น
- **ลบผู้ใช้**: เข้าเว็บด้วยบัญชี admin → แท็บ "จัดการผู้ใช้" → กด "ลบ" ที่ผู้ใช้คนนั้น
- **ดูรายชื่อผู้ใช้ทั้งหมด**: ดูได้ในแท็บ "จัดการผู้ใช้" โดยตรง หรือรันผ่าน CLI:
  ```bash
  wrangler d1 execute care-billing-db --remote --command="SELECT username,display_name,role,branch_id FROM users"
  ```

## หมายเหตุสำคัญ

- Session cookie อยู่ได้ 7 วันแล้วต้องล็อกอินใหม่ (ปรับได้ที่ `SESSION_MAX_AGE` ใน `src/index.js`)
- การบันทึกข้อมูลของ staff ทำงานแบบ "แทนที่ข้อมูลเฉพาะในสาขาตัวเอง" ทุกครั้งที่กดบันทึก — ปลอดภัยต่อข้อมูลสาขาอื่นเพราะฝั่งเซิร์ฟเวอร์ตรวจสอบ branch_id ทุกครั้งก่อนเขียนจริง ไม่ใช่แค่ซ่อนปุ่มที่หน้าเว็บ
- API จัดการผู้ใช้ (`/api/users`, `/api/users/:id`, `/api/users/:id/reset-password`) ตรวจสอบสิทธิ์ admin ทุกครั้งที่ฝั่งเซิร์ฟเวอร์ และไม่ส่ง password_hash/salt กลับมาที่ฝั่งเว็บเด็ดขาด
- ปุ่ม "ดาวน์โหลด JPG" โหลดไลบรารี `html2canvas` จาก CDN (cdnjs.cloudflare.com) จึงต้องมีอินเทอร์เน็ตตอนใช้งานปุ่มนี้ (ปุ่มพิมพ์/บันทึก PDF เดิมยังใช้งานได้ปกติแม้ไม่มีอินเทอร์เน็ต)
